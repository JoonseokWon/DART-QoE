# DART-QoE

공시 재무제표와 주석을 이용해 인수 검토 대상 기업의 **이익 지속성, 현금전환과 운전자본**을 살펴보는 PoC입니다. 자동으로 정상화 이익이나 기업가치를 판단하지 않고, 거래 검토자가 원문을 확인할 항목과 계산 근거를 빠르게 찾도록 설계했습니다.

## 포트폴리오에서의 역할

`한화–대우조선해양 M&A 사례 해석 → DART-QoE로 실제 거래 검토 방식 구현 → DART-OT로 이자비용 감사 검토`의 흐름을 만듭니다. 한화오션 사례에서 드러난 유동성·운전자금·재무구조 이슈를 정량 검토로 연결하되, 실제 거래실사 경험으로 표현하지 않습니다.

## MVP 범위

- 제조업 상장사, 최근 3개 사업연도, 연결재무제표 우선
- 매출·영업이익·순이익·영업현금흐름과 현금전환
- 매출채권·재고·매입채무 회전일수, 순운전자본
- 차입금·사채, 현금, 선택적 리스부채를 이용한 순차입금
- 정상화 조정 **후보** 5종: 자산 처분, 손상·충당부채, 소송·재해, 정부보조금, 비영업·중단영업·기타손익
- 공시·계정·원문·산식·자동 추출·사용자 조정 사유를 남기는 Excel

EBITDA와 순차입금/EBITDA는 감가상각비·상각비의 안정적인 연결 기준 추출을 추가한 뒤 확장합니다. 현재는 영업이익 기준 MVP입니다.

## 실행

Windows에서는 `DART-QoE.exe` 하나만 실행하면 됩니다. 실행파일은 Python, Node.js와 Excel 생성 모듈을 모두 포함하며, 결과 파일은 실행파일 옆의 `outputs` 폴더에 저장됩니다. 실행 중에는 서버 콘솔 창이 함께 열리며, 이 창을 닫으면 프로그램도 종료됩니다.

```powershell
python app.py
```

브라우저에서 `http://127.0.0.1:8765`가 열립니다. OpenDART API 키와 회사명, 분석기간을 입력하거나 API 키 없이 데모 Excel을 만들 수 있습니다.

이 Codex 작업환경에서는 번들 Node.js와 `@oai/artifact-tool`로 Excel을 생성합니다. 다른 환경에서는 `DART_QOE_NODE`와 `NODE_PATH`가 필요할 수 있습니다.

### 실행파일 빌드

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build_exe.ps1
```

빌드 결과는 프로젝트 루트의 `DART-QoE.exe`입니다.

## Excel 구성

- `Cover`: 목적, 한계, 분석 범위와 사용 순서
- `Source Data`: DART 연결 재무제표 추출값과 사용자 메모
- `QoE Summary`: 보고이익, 현금전환, 순차입금
- `Working Capital`: 회전일수, 현금전환주기, 순운전자본
- `Review Candidates`: 원문 발췌와 사용자 조정 여부·사유
- `Audit Trail`: 사용 공시, 접수번호, 원문 URL, 계정과 산식
- `Checks`: 데이터 완전성, 오류와 제한사항

## 해석상 주의

- 자동 탐지한 항목을 정상화 조정으로 확정하지 않습니다.
- 계정명만으로 일회성을 판단하지 않습니다.
- 회전일수의 분모·기간과 연결/별도 기준을 반드시 확인합니다.
- 추출 오류를 조용히 제외하지 않고 `Checks`에 남깁니다.
- 결과는 PoC이며 회계·투자 자문 또는 실제 거래실사 결과가 아닙니다.
